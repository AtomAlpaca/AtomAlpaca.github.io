写在前面

这是我个人写竞赛代码是的代码规范，主要根据个人习惯，也参考了_rqy的代码规范。
请谨慎运用于工程代码。

缩进、空格与换行

每个代码块使用一个 tab 缩进，如果 tab 键无法使用，则使用四个空格缩进
规定一个 tab 为四个空格宽

大括号应当换行
表达式过长的时候可以换行，运算符在行尾。
函数不能写到一行内

需要加空格的地方：

  1. 除 , 的二元运算符两端；
  2. , 的右侧；
  3. if 、 while 、 for 等关键字与括号之间；
  4. 表示指针的 * 的两侧、 表示引用的 & 的左侧。

命名空间

命名空间应置于头文件之后，主函数之前
考虑到竞赛代码的特殊性，不应当在任何函数内使用命名空间
为了避免函数名和变量名冲突，使用命名空间时应使用 using example::func 而不是 using namespace example

函数

主函数返回值为 int，末尾的 return 0 不可以省略。主函数可以传入参数，但在实际比赛中为了节省时间建议主函数不写参数。
短函数不能写到一行。

命名

可以使用约定俗成的单字符变量名，如 T 表示数据组数、d表示距离，etc.；如果没有约定俗成的单字符变量名可用，变量名应采用驼峰命名法，开头字母小写。

函数名首字母应当大写，而结构体、类和命名空间名首字母应当大写。
